# 任务分解（TASK）

## 现状记录
当前问题：参考音频会出现“只读后半段 + 音色不相似”
主要原因假设：`txt_smp` 作为前缀提示导致续写；参考文本未严格对齐
现有结果：开启参考音频清洗后可发声，但相似度仍偏低
搁置说明：等待用户提供逐字转写与目标文本确认后继续

## 任务清单

### [x] 基线验证（随机音色可发声）
目标：确认模型可正常发声
输入：随机音色 + 短文本
输出：可听的 mp3
完成标准：音频时长正常且非静音
风险点：CPU 推理耗时较长

### [x] 参考音频裁剪清洗接入
目标：降低参考音频导致的近静音输出
输入：参考音频文件 + 静音阈值
输出：清洗后的参考音频用于采样
完成标准：批量输出可听且不为 0 秒
风险点：过度裁剪可能损失音色信息

### [ ] 参考文本逐字对齐校验
目标：避免“只读后半段”问题
输入：参考音频逐字转写 + 目标文本
输出：对齐后的 `参考文本` 字段
完成标准：生成内容覆盖完整目标文本
风险点：转写误差会导致音色漂移

### [ ] 多次采样择优策略
目标：提升音色相似度
输入：相同文本多次采样
输出：可听感更接近参考的版本
完成标准：主观听感明显更接近参考音色
风险点：采样开销较大

### [ ] 文本拆分策略验证
目标：确认拆分对“续写”影响
输入：同一文本在 split_text=开/关下生成
输出：对比结果说明
完成标准：确定是否需要关闭拆分
风险点：长文本可能超出模型长度
